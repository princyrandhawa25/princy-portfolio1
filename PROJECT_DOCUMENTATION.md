# Princy Portfolio Project Documentation

This project is a React portfolio website built with Create React App and Tailwind CSS. It stores most editable content in JSON or Markdown files under `public/portfolio-data`, with matching copies under `portfolio-data`.

## Project Structure

- `src/` contains React pages, components, styles, and imported image assets.
- `src/pages/` contains route-level pages such as Home, Journals, Conferences, Gallery, Awards, and Academics.
- `src/components/` contains reusable UI sections.
- `src/assets/` contains images used by the app.
- `src/assets/gallery_images/` contains images shown on the Gallery page.
- `src/assets/carousal_images/` contains homepage carousel images.
- `public/portfolio-data/` contains live data files loaded by the website in the browser.
- `portfolio-data/` is a matching data copy kept in the repository.
- `build/` is generated by `npm run build` and should not be edited manually.

## Deploy to GitHub Pages

This project already has GitHub Pages deployment scripts:

```json

"deploy": "gh-pages -d build"
```

To deploy:


## Common Content Updates

For most content updates, edit both the file in `public/portfolio-data` and the matching file in `portfolio-data`.

The `public/portfolio-data` copy is the one loaded by the live website. The `portfolio-data` copy is kept as a project data backup/source copy.

### Journals

Edit both files:

```text
public/portfolio-data/journals.md
portfolio-data/journals.md
```

Add new journal entries near the top and keep the numbering clean.

### Conferences

Edit both files:

```text
public/portfolio-data/conferences.json
portfolio-data/conferences.json
```

Add new conference records inside `conference_presentations`.

### Awards and Achievements

Awards are stored in:

```text
public/portfolio-data/awards.json
portfolio-data/awards.json
```

Achievements are stored in:

```text
public/portfolio-data/achievements.json
portfolio-data/achievements.json
```

### Academics and Courses Taught

Edit both files:

```text
public/portfolio-data/academics.json
portfolio-data/academics.json
```

Courses are listed under `courses_taught`.

### Gallery Images

Add gallery images to:

```text
src/assets/gallery_images/
```

The Gallery page automatically loads images from this folder.

To make an image appear first, use a filename prefix such as:

```text
00-image-name.jpeg
01-image-name.jpeg
```

### Homepage Carousel Images

Add carousel images to:

```text
src/assets/carousal_images/
```

Malaysia carousel images are stored in:

```text
src/assets/carousal_images/malaysia/
```

The homepage prioritizes images from the `malaysia` folder before other carousel images.

## Step-by-Step Update Process

Follow this process whenever making changes.

1. Pull the latest code before editing:

```bash
git pull origin main
```

2. Check current changed files:

```bash
git status
```

3. Make the required updates.

```text
public/portfolio-data/
```


```bash
git add .
```

11. Push to GitHub:

```bash
git push origin main
```


13. Open the live website and confirm the change is visible:


## Before Pushing Checklist

- The site runs locally with `npm start`.
- The changed page was checked in the browser.
- JSON files are valid and do not contain trailing commas.
- New images are inside the correct `src/assets` folder.
- The production build passes with `npm run build`.
- `git status` shows only the files intended for the update.

## Troubleshooting

If `npm start` fails, run:

```bash
npm install
```

Then try again:

```bash
npm start
```

If `npm run build` fails, read the terminal error and fix the file mentioned in the error.

If Git says the branch is not `main`, check the active branch:

```bash
git branch
```

Then push using that branch name:

```bash
git push origin branch-name
```

If deployment does not update immediately, wait a few minutes and refresh the browser. GitHub Pages can take a short time to publish.
